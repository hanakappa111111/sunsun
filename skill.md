# プロのWeb制作ワークフロー & 手順書

## 概要

このスキルドキュメントは、プロフェッショナルなWebサイト（HP）制作における標準的なワークフローと手順を定義する。品質の高い成果物を効率的に納品するための指針。

---

## Phase 0: 事前ヒアリング・要件定義

### ヒアリング項目

| カテゴリ | 確認事項 |
|----------|---------|
| **目的** | サイトの目的（集客、認知向上、EC、採用、情報発信） |
| **ターゲット** | ユーザー属性（年齢、性別、職業、デバイス傾向） |
| **競合** | 参考サイト3〜5件（良い点・悪い点のヒアリング） |
| **コンテンツ** | ページ数、文章・画像の支給有無 |
| **機能** | フォーム、CMS、多言語、予約システム等 |
| **スケジュール** | 公開希望日、マイルストーン |
| **予算** | 制作費・保守費の上限 |
| **運用体制** | 公開後の更新担当者・頻度 |

### アウトプット
- ヒアリングシート（記録）
- 要件定義書（What / Why / Who / When / How）
- スコープ合意書（追加作業の線引き）

## Phase 1: サイト設計（IA・情報設計）

### サイトマップ作成
- 全ページのリスト化と階層構造の定義
- ナビゲーション構造の決定（グローバル・ローカル・フッター）
- ツール: FigJam / Whimsical / draw.io

### ワイヤーフレーム作成
- 各ページのレイアウト骨格を設計（デザイン前に構造確定）
- PCとスマホ両方を作成
- コンテンツの優先順位・視線の流れを意識
- ツール: Figma / Adobe XD / Balsamiq

### コンテンツ計画
- コピー（テキスト）の担当を確認（支給 or ライティング込み）
- 画像・動画素材の調達方法（支給 / ストック / 撮影）
- SEOキーワード選定と各ページへの割り当て

## Phase 2: デザイン

### デザインシステム策定
- カラーパレット（メイン・サブ・アクセント・背景・テキスト）
- タイポグラフィ（フォントファミリー、サイズ階層、行間）
- スペーシングルール（8px グリッド推奨）
- コンポーネント定義（ボタン・カード・フォーム・アイコン）

### デザインカンプ制作順序
1. トップページ（最重要 → クライアント確認・方向性合意）
2. 下層ページテンプレート（共通パーツの流用設計）
3. 全ページ展開

### デザインチェックリスト
- [ ] ブランドガイドラインとの整合性
- [ ] 文字の可読性（コントラスト比 4.5:1 以上 / WCAG AA準拠）
- [ ] ホバー・アクティブ状態の定義
- [ ] 404・エラーページのデザイン
- [ ] ローディング状態の考慮
- [ ] OGP画像のデザイン

### クライアント確認フロー
1. Figmaリンク or PDF で共有
2. フィードバック収集（ラウンド制: 修正は2ラウンドまで）
3. サイン・オフ（書面または電子での承認）

## Phase 3: コーディング

### 環境構築

```bash
# プロジェクト初期化（Next.js の場合）
npx create-next-app@latest project-name --typescript --tailwind --eslint

# Vite + React の場合
npm create vite@latest project-name -- --template react-ts

# 静的サイトの場合
mkdir project-name && cd project-name
git init
npm init -y
```

### ディレクトリ構成（標準）

```
project/
├── public/          # 静的アセット（favicon, OGP画像）
├── src/
│   ├── assets/      # 画像・フォント・動画
│   ├── components/  # 再利用UIコンポーネント
│   ├── layouts/     # ページレイアウト
│   ├── pages/       # ページコンポーネント
│   ├── styles/      # グローバルCSS / CSS変数
│   └── utils/       # ヘルパー関数
├── .gitignore
├── package.json
└── README.md
```

### コーディング標準

**HTML**
- セマンティックHTML（`header`, `main`, `nav`, `section`, `article`, `footer`）
- `alt`属性の適切な記述
- `lang`属性の設定

**CSS**
- CSS変数（カスタムプロパティ）でデザイントークン管理
- モバイルファースト（`min-width`のメディアクエリ）
- BEM記法 or CSS Modules / Tailwind CSS

**JavaScript / TypeScript**
- ES2020+ の構文
- コンポーネント単位でのファイル分割
- 非同期処理は `async/await`

### ブレークポイント（推奨）

| 名称 | 幅 |
|------|----|
| Mobile | ~ 767px |
| Tablet | 768px ~ 1023px |
| Desktop | 1024px ~ |
| Wide | 1440px ~ |

### パフォーマンス要件
- LCP（最大コンテンツ描画）: 2.5秒以内
- CLS（累積レイアウトシフト）: 0.1以下
- FID / INP: 200ms以下
- 画像: WebP形式、`loading="lazy"`、`width/height`属性必須

## Phase 4: コンテンツ実装・SEO

### SEO基本設定
- `<title>`: 30〜35字（ブランド名 | ページ名）
- `<meta description>`: 120〜140字
- OGP（`og:title`, `og:description`, `og:image`, `og:url`）
- canonical URL の設定
- sitemap.xml の生成
- robots.txt の設定
- 構造化データ（JSON-LD）: 組織・パンくず・FAQなど

### アナリティクス設定
- Google Analytics 4（GA4）の導入
- Google Search Console への登録・サイトマップ送信
- 目標・コンバージョン設定

## Phase 6: 公開（デプロイ）

### 事前確認
- [ ] 本番用環境変数の設定
- [ ] Basic認証の解除
- [ ] `.env` ファイルがリポジトリに含まれていないことを確認
- [ ] noindex タグが外れていることを確認
- [ ] ステージング環境での最終確認

### DNS・サーバー設定
- ドメイン取得・移管確認
- SSL証明書（Let's Encrypt または有料）
- HTTPSリダイレクト設定
- www あり/なしのどちらかに統一

### デプロイ手順（例）

```bash
# ビルド
npm run build

# Vercel デプロイ（Next.js）
vercel --prod

# FTP アップロードの場合
# lftp / FileZilla などで /public_html へ転送

# GitHub Pages
git push origin main  # GitHub Actions で自動デプロイ
```

### 公開後確認
- [ ] 本番URLでの全ページ動作確認
- [ ] Search Console のURL検査
- [ ] GA4 のリアルタイムでのアクセス確認

## コミュニケーション原則

- **進捗報告**: 週1回以上の定期報告
- **修正ラウンド**: デザイン2ラウンド・コーディング2ラウンドまでを標準スコープとする
- **変更管理**: スコープ外追加は見積もりを出してから着手
- **ファイル命名**: `YYYYMMDD_案件名_v1.0` の形式で版管理
